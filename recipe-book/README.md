# The Reinhardt Family Recipe Box

A single-page recipe box: `index.html` renders a 3D recipe box whose divider
tabs open galleries of scanned recipe cards. Everything it needs lives in this
folder — no server code, no database.

## Folder layout

```
recipe-book/
├── index.html      the whole app (markup, styles, script)
├── recipes.json    manifest generated by build.py — do not edit by hand
├── build.py        regenerates recipes.json + optimized images
├── images/         web-size scans (~1600px) shown in the lightbox
└── thumbs/         small scans (~560px) shown in the category grids
```

The original scans live in `../Recipes/<Category>/…` and are never modified.
File naming convention: `<Prefix> - <Title>.jpeg`, with multi-page recipes as
`<Title> (1).jpeg`, `<Title> (2).jpeg`, … Underscores become apostrophes in
displayed titles. Each subfolder of `Recipes/` becomes a divider in the box.

## Adding or changing recipes

1. Drop the scan(s) into the right `Recipes/<Category>/` folder using the
   naming convention above (a new folder = a new category divider).
2. Run `py build.py` from this folder. It converts only new/changed scans and
   rewrites `recipes.json`.
3. Re-upload `recipes.json` plus the new files in `images/` and `thumbs/`.

## Deploying next to WordPress (self-hosted)

The app deploys as a static folder alongside WordPress — not inside it.

1. Connect to your host with FTP or the hosting control panel's file manager.
2. In the web root (the folder holding `wp-admin/`, `wp-content/`,
   `wp-config.php` — usually `public_html/`), create a folder named
   `recipe-box`.
3. Upload the **contents** of this folder into it: `index.html`,
   `recipes.json`, `images/`, `thumbs/` (`build.py` optional but harmless).
4. Visit `https://your-site.com/recipe-box/` — the page should work
   immediately. WordPress only routes URLs that don't match real files or
   folders, so a real `recipe-box/` folder is served directly and nothing
   about WordPress needs to change.
5. Link it from the site: WP Admin → Appearance → Menus (or the theme
   customizer) → add a **Custom Link** to `/recipe-box/` labeled
   "Family Recipes".

### Notes

- If your host serves a 403 for the folder, the `index.html` default-document
  rule is off for that directory; most Apache/cPanel hosts honor it out of
  the box.
- The page must be served over HTTP(S) — it fetches `recipes.json`, which
  browsers block on `file://` URLs.
- All paths inside the app are relative, so the folder can be renamed or
  nested anywhere (e.g. `/family/recipes/`) without editing anything.
